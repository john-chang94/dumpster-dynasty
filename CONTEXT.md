# Dumpster Dynasty MVP Roadmap and Asset Plan

A detailed, no-code planning document for building **Dumpster Dynasty**, a cute raccoon idle-builder with light active play, cozy base progression, collectible raccoons, loot discovery, and mobile-friendly monetization.

---

## 1. Game Vision

**Dumpster Dynasty** is a cozy idle-builder where players grow a hidden raccoon settlement by scavenging food, junk, scrap, and shiny treasures from nearby city blocks.

The player is the leader of a raccoon crew building a charming trash kingdom behind the city. By night, raccoons scavenge. By day, the player sorts loot, upgrades buildings, unlocks new locations, and recruits more raccoons.

### Core Fantasy

You are building a cozy, mischievous raccoon empire out of discarded city junk.

The tone should feel:

- Cute
- Mischievous
- Cozy-chaotic
- Collectible
- Playful, not gritty
- Lightly strategic without being overwhelming

### Genre Positioning

The best positioning is:

**Cozy idle-builder + raccoon collector**

Avoid positioning it as a full action game, combat game, runner, or raid battler during the MVP. The strongest early value is in progression, collection, and personality.

---

## 2. MVP Design Summary

The MVP should prove the core loop with enough charm to feel like a real game, but without overbuilding complex systems.

### MVP Feature Set

Include:

- One main Home/Base screen
- Three scavenging zones
- Four raccoon classes
- Three resources
- Five core buildings
- Offline earnings
- Collection book
- Light active gameplay through a choice-based Active Scavenge system
- Basic tutorial
- Basic sound effects and music
- Local save/load
- Early monetization foundation

Do not include yet:

- Multiplayer
- Guilds
- Deep crafting
- Complex combat
- Procedural maps
- Dozens of animations per character
- Live events
- Battle pass
- Deep narrative system
- Real-time raid gameplay

## 3. Core Gameplay Loop

### Main Loop

1. Open the app.
2. Collect offline loot.
3. Assign raccoons to scavenging zones.
4. Claim food, scrap, shinies, and loot items.
5. Spend resources on buildings and upgrades.
6. Improve scavenging efficiency.
7. Unlock new zones, raccoons, and collection entries.
8. Optionally play an Active Scavenge encounter.
9. Leave and return later.

### Short Session Loop

Designed for one-minute mobile play sessions:

1. Open app.
2. Claim offline rewards.
3. Start or claim scavenging runs.
4. Buy one to three upgrades.
5. Check collection progress.
6. Optionally play one quick Active Scavenge.
7. Close app.

### Medium Session Loop

Designed for two to five minutes:

1. Claim offline rewards.
2. Optimize raccoon assignments.
3. Upgrade buildings.
4. Discover collection entries.
5. Complete a daily objective.
6. Play one or two Active Scavenge encounters.

### Longer Session Loop

Designed for five to ten minutes:

1. Review all active zones.
2. Upgrade multiple buildings.
3. Compare raccoon bonuses.
4. Check collection gaps.
5. Play Active Scavenge.
6. Visit shop or rewards.
7. Plan next unlock.

## 4. Resources

The MVP should use only three core resources.

### Food

Food is the basic progression resource.

Used for:

- Recruiting raccoons
- Basic upgrades
- Snack Stash upgrades
- Possibly entering low-risk Active Scavenge encounters

### Scrap

Scrap is the main building and upgrade resource.

Used for:

- Building upgrades
- Sort Station upgrades
- Nest Pile upgrades
- Training Crate upgrades
- Base expansion later

### Shinies

Shinies are rare premium-feeling resources earned slowly through gameplay and optionally purchased later.

Used for:

- Rare unlocks
- Cosmetic items
- Speed-ups
- Premium-feeling upgrades
- Special rewards

Important: Shinies should not be paid-only. Players should earn them slowly through gameplay so the economy feels fair.

### Optional Future Resources

Not recommended for MVP, but possible later:

- Batteries
- Rare toys
- Golden trash
- Event tokens
- Decoration materials
- Recycling badges

## 5. Raccoon Classes

The MVP uses four role-based raccoon classes. Each class should affect strategy without creating too much complexity.

### Scout Raccoon

Role: faster scavenging and speed-based checks.

Visual identity:

- Green backpack
- Green bandana
- Alert expression
- Adventurous silhouette

Gameplay bonus:

- Faster scavenging timers
- Better performance in speed-based Active Scavenge choices

### Hauler Raccoon

Role: carries more resources, especially scrap.

Visual identity:

- Orange scarf or bandana
- Bigger backpack or tool belt
- Stronger, sturdier body shape
- Carries crates or scrap

Gameplay bonus:

- Increased scrap rewards
- Increased total haul from zones
- Better performance in heavy-carry choices

### Sniffer Raccoon

Role: finds rare loot and collection items.

Visual identity:

- Goggles
- Magnifying glass
- Teal scarf
- Curious detective vibe

Gameplay bonus:

- Higher rare item chance
- Better collection discovery rate
- Better performance in search/sniff choices

### Sneak Raccoon

Role: improves shiny drop chance and risk avoidance.

Visual identity:

- Purple scarf or hood
- Lower sneaking posture
- Mischievous but still cute

Gameplay bonus:

- Better shiny drop chance
- Lower risk in Active Scavenge encounters
- Better success in stealth/grab choices

## 8. Active Scavenge System

The original raid concept is fun, but a real-time mini-runner is too risky for MVP. For the MVP, use a choice-based Active Scavenge system instead.

### Recommended Name Options

- Night Run
- Scavenge Run
- Sneak Mission
- Trash Dash
- Loot Run

For MVP, **Night Run** or **Active Scavenge** fits best.

### MVP Active Scavenge Flow

1. Player taps Night Run.
2. Player chooses one raccoon.
3. The game presents a short encounter.
4. Player chooses from two or three actions.
5. Each choice checks raccoon class strengths.
6. Player receives a success, partial success, or fail result.
7. Rewards are granted.

### Example Encounter Types

Security Light:

- Hide for low risk and small reward
- Dash for medium risk and medium reward
- Grab shiny first for high risk and high reward

Barking Dog:

- Distract
- Sneak past
- Retreat safely

Suspicious Trash Bag:

- Open carefully
- Sniff first
- Grab quickly

Vending Machine:

- Shake it
- Search underneath
- Ignore it

Rival Raccoon:

- Share loot
- Race to the stash
- Sneak around

Locked Dumpster:

- Use scrap tool
- Call Hauler
- Search another bin

### Why This Works for MVP

It adds:

- Player agency
- Risk/reward decisions
- Personality
- Class strategy
- Repeatability

It avoids:

- Real-time input complexity
- Collision detection
- Performance issues
- More animation requirements
- Difficulty tuning problems

### Future Upgrade

Later, this can become a real-time lane-based runner called **Trash Dash**, using the same raccoon stats and reward systems.

Shows:

- Raccoon village/base

- Buildings

- Resource rewards should use icons and dynamic text.

## 10. Monetization Plan

Use a fair hybrid model.

### Rewarded Ads

Good uses:

- Double offline rewards
- Speed up a scavenge timer
- Get one extra Active Scavenge attempt
- Bonus daily gift
- Reroll reward result

Avoid:

- Forced ads after every action
- Ads interrupting the core loop
- Making rewarded ads mandatory for progress

### In-App Purchases

Recommended products:

- Starter Trash Pack
- Small Shiny Pouch
- Medium Shiny Hoard
- Big Shiny Stash
- Remove Ads
- Cosmetic skin packs

### Cosmetics

Best long-term IAP category.

Future cosmetic ideas:

- Raccoon hats
- Raccoon scarves
- Base themes
- Dumpster skins
- Seasonal decorations
- Rare raccoon variants

### Battle Pass

Not recommended for MVP.

Possible future feature:

- Trash Pass
- Daily and weekly objectives
- Free rewards track
- Premium cosmetic track

## 12. Architecture Strategy

### Local-First Gameplay

The game should not depend on Firestore for every tap. Immediate gameplay should happen locally.

Use local state for:

- Resources
- Buildings
- Raccoons
- Active runs
- Collection state
- Last claim time
- Offline rewards

Use Firebase for:

- Cloud save backup
- User identity
- Purchase records
- Economy config later
- Analytics and remote tuning

### Sync Strategy

Sync important snapshots:

- On app launch
- When app backgrounds
- After purchases
- After major milestones
- Periodically during longer sessions

### Config Strategy

For MVP, game configs can live locally as data files.

Later, move balancing values to remote config or Firestore:

- Zone rewards
- Building costs
- Drop rates
- Timers
- Event rewards
- Shop offer tuning

## 17. Global Asset Rules

### File Format

Most game assets should be:

- PNG
- Transparent background
- Clean alpha edges
- High-resolution master exports
- No baked-in functional text
- Consistent lighting and outline thickness

Large backgrounds may be:

- PNG
- WebP if compression is needed
- Separate layered images if parallax or foreground effects are desired

### Text Rules

Functional text should be rendered dynamically in code.

Do not bake in:

- Button labels
- Prices
- Resource amounts
- Timers
- Upgrade costs
- Quest text
- Level numbers
- Reward values
- Zone names if localization may be needed

Text can be baked into:

- Game logo
- Purely decorative signs
- Marketing screenshots
- Static promotional images

### Sprite Sheet Rules

For production sprite sheets:

- No labels
- No section headers
- No documentation text
- No title banners
- Transparent background
- Generous padding between assets
- Consistent grid when possible
- No shadows outside each cell
- Do not let assets touch

### Atlas Rules

Recommended atlas groups:

- Characters
- Loot and resources
- Buildings and props
- UI core
- Effects
- Tutorial

Do not put large backgrounds into small sprite atlases. Keep backgrounds separate unless there is a strong reason to pack them.

## 33. Sprite Production Workflow

Use two different types of asset sheets:

### Reference Sheets

These can include:

- Labels
- Section headers
- Documentation text
- Notes
- Grouping borders

Purpose:

- Human reference
- Planning
- Naming guide

Do not use these directly for atlas slicing.

### Production Sheets

These should include:

- Only the assets
- No labels
- No text
- No headers
- No borders
- Transparent background
- Equal cell grid when possible
- Generous padding

Purpose:

- Slicing
- Packing
- Atlas generation

### Best Workflow

1. Generate a labeled reference sheet.
2. Generate or create an unlabeled production sheet.
3. Slice the production sheet using grid or rectangular slicing.
4. Rename each exported sprite using the reference sheet names.
5. Pack individual sprites with TexturePacker.
6. Export atlas PNG and JSON metadata.
7. Load atlas and metadata in the app.

---

## 34. TexturePacker Guidance

TexturePacker usually packs already-separated sprite files. If your version only offers grid or rectangular splitting, use clean production sheets.

### For Grid Splitting

Best for:

- Character frames
- Loot icons
- Resource icons
- Common UI icons
- FX icons

Requirements:

- Equal cell size
- Assets centered inside cells
- No labels
- Transparent background

Example setup:

- Four columns by five rows
- Each cell 256 by 256
- Asset centered in each cell

### For Rectangular Splitting

Best for:

- Long buttons
- Panels
- Progress bars
- Banners
- Large UI frames
- Irregular assets

### Recommended Packing Settings

Use:

- JSON metadata format
- PNG texture format
- RGBA8888 pixel format
- Trim enabled for static assets
- Shape padding around 4 pixels
- Border padding around 4 pixels
- Extrude around 1 to 2 pixels
- Rotation off
- Maximum texture size 2048 by 2048
- Multipack on if needed

### Character Animation Cropping Rule

For character animations:

- Equal-sized cells are easiest.
- Keep each raccoon centered.
- Preserve consistent anchor/pivot across frames.
- Avoid trimming if it causes animation jitter, unless metadata offsets are handled properly.

For static icons and UI:

- Trim tightly to save atlas space.

## 37. Final MVP Recommendation

Build **Dumpster Dynasty** as a cozy idle-builder first, not as an action game.

The highest-value MVP features are:

- Cute raccoon collection
- Satisfying offline rewards
- Simple zone assignment
- Clear building upgrades
- Collection book progress
- Choice-based Active Scavenge
- Strong visual identity

The real-time raid/runner idea should become a later feature after the core game is already fun and stable.

Recommended final MVP tagline:

**Send your raccoon crew into the city, collect ridiculous loot, and grow the coziest trash kingdom ever.**
